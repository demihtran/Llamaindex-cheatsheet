LI

Framework Overview

Data framework for LLM apps and RAG
Khái niệm LlamaIndex kết nối private data với LLM qua loading, indexing, retrieval, querying.
Khi dùng RAG, chat over docs, structured extraction, agents dùng query engines/tools.
Install pip install -U llama-index hoặc package tách nhỏ.
Core Flow Load -> parse -> index -> retrieve -> synthesize -> evaluate.
pip install -U llama-index
pip install -U llama-index-llms-openai
pip install -U llama-index-embeddings-openai

from llama_index.core import (
    VectorStoreIndex,
    SimpleDirectoryReader,
    Settings,
)

documents = SimpleDirectoryReader("data").load_data()
index = VectorStoreIndex.from_documents(documents)
query_engine = index.as_query_engine()
response = query_engine.query("Summarize the docs")

Best: bắt đầu bằng pipeline RAG đơn giản, sau đó tối ưu từng lớp retrieval/synthesis.

Mistake: coi LlamaIndex chỉ là vector DB wrapper; phần query engine/synthesis cũng rất quan trọng.

SET

Settings

Global configuration for models and parsing
Khái niệm Global settings thay thế ServiceContext cũ cho LLM, embedding, splitter, callbacks.
Khi dùng Đặt default model/embedding/chunking cho toàn app.
Import from llama_index.core import Settings
Fields llm, embed_model, text_splitter, chunk_size.
from llama_index.core import Settings
from llama_index.llms.openai import OpenAI
from llama_index.embeddings.openai import OpenAIEmbedding
from llama_index.core.node_parser import SentenceSplitter

Settings.llm = OpenAI(model="gpt-4o-mini")
Settings.embed_model = OpenAIEmbedding(
    model="text-embedding-3-small"
)
Settings.text_splitter = SentenceSplitter(
    chunk_size=800,
    chunk_overlap=120,
)

Best: cấu hình model/embedding một chỗ, pin versions, dùng env vars cho keys.

Mistake: trộn nhiều embedding models trong cùng index hoặc dùng ServiceContext cũ cho code mới.

LLM

LLMs & Embeddings

Model integrations for generation and vectors
Khái niệm LLM generate/synthesize; embedding model biến text thành vectors.
Khi dùng Query engine, response synthesis, agents, semantic retrieval.
Constructors OpenAI(model=...), OpenAIEmbedding(model=...)
Methods complete, chat, stream_complete, embedding qua index.
from llama_index.llms.openai import OpenAI
from llama_index.embeddings.openai import OpenAIEmbedding

llm = OpenAI(model="gpt-4o-mini", temperature=0.1)
embed_model = OpenAIEmbedding(model="text-embedding-3-small")

completion = llm.complete("Explain RAG in one sentence.")
chat = llm.chat([
    {"role": "system", "content": "Be concise."},
    {"role": "user", "content": "What is LlamaIndex?"},
])

Settings.llm = llm
Settings.embed_model = embed_model

Best: chọn embedding model trước khi build index và giữ nguyên khi query.

Mistake: đổi embedding model sau khi index đã build mà không re-index.

LD

Loading Data

Readers, connectors, LlamaHub, LlamaParse
Khái niệm Readers/loaders chuyển file, web, DB, APIs thành Document.
Khi dùng Ingest dữ liệu trước khi parse/chunk/index.
Import SimpleDirectoryReader, connector packages từ LlamaHub.
Methods load_data(), connector-specific options.
from llama_index.core import SimpleDirectoryReader

documents = SimpleDirectoryReader(
    input_dir="data",
    recursive=True,
    required_exts=[".pdf", ".md", ".txt"],
).load_data()

for doc in documents:
    print(doc.id_, doc.metadata)

# LlamaParse for complex PDFs/tables:
# pip install llama-parse
# from llama_parse import LlamaParse

Best: giữ metadata như source, page, tenant_id để filter/cite/debug.

Mistake: ingest dữ liệu thô nhiều noise hoặc bỏ metadata nguồn.

DOC

Documents & Nodes

Core data units before and after parsing
Document Raw/semi-raw text + metadata từ loader.
Node Chunk có ID, text, metadata, relationships, embedding.
Import Document, TextNode
Methods get_content(), set_content(), metadata fields.
from llama_index.core import Document
from llama_index.core.schema import TextNode

doc = Document(
    text="LlamaIndex builds RAG pipelines.",
    metadata={"source": "notes.md", "section": "intro"},
)

node = TextNode(
    text="A node is a chunk for retrieval.",
    metadata={"source": "notes.md"},
)

print(doc.text)
print(node.get_content())

Best: Document cho ingestion, Node cho indexing/retrieval; metadata nên nhất quán.

Mistake: chunk mất source/page khiến response không cite được.

NP

Node Parsers

Split documents into retrieval-ready nodes
Khái niệm Node parser/chunker tạo nodes từ documents.
Khi dùng PDF/docs dài, code, markdown, corpus cần retrieval.
Constructor SentenceSplitter(chunk_size, chunk_overlap)
Methods get_nodes_from_documents(), transformations trong ingestion.
from llama_index.core.node_parser import SentenceSplitter

splitter = SentenceSplitter(
    chunk_size=800,
    chunk_overlap=120,
)

nodes = splitter.get_nodes_from_documents(documents)

index = VectorStoreIndex(nodes)

Best: tune chunking bằng câu hỏi thật và đánh giá retrieval recall.

Mistake: chunk quá nhỏ mất context, quá lớn làm embedding/retrieval kém.

ING

Ingestion Pipeline

Transform, parse, extract metadata, embed, cache
Khái niệm Pipeline chạy chuỗi transformations từ documents sang nodes đã sẵn sàng index.
Khi dùng Production ingestion, caching, metadata extraction, batch processing.
Import IngestionPipeline, SentenceSplitter, embedding transforms.
Methods run(), arun(), optional cache/vector_store.
from llama_index.core.ingestion import IngestionPipeline
from llama_index.core.node_parser import SentenceSplitter
from llama_index.embeddings.openai import OpenAIEmbedding

pipeline = IngestionPipeline(
    transformations=[
        SentenceSplitter(chunk_size=800, chunk_overlap=120),
        OpenAIEmbedding(model="text-embedding-3-small"),
    ],
)

nodes = pipeline.run(documents=documents)

# You can add metadata extractors, custom transforms,
# cache, or a vector_store for production ingestion.

Best: build ingestion riêng query path; cache transformations đắt tiền.

Mistake: embed lại toàn bộ corpus mỗi lần app start hoặc không version pipeline.

IDX

VectorStoreIndex

Most common index for semantic search and RAG
Khái niệm Index embeds nodes/documents và lưu vào vector store.
Khi dùng RAG trên docs, semantic search, chat over private data.
Constructors from_documents(), VectorStoreIndex(nodes)
Methods as_query_engine, as_retriever, as_chat_engine, insert.
from llama_index.core import VectorStoreIndex

index = VectorStoreIndex.from_documents(
    documents,
    show_progress=True,
)

query_engine = index.as_query_engine(
    similarity_top_k=4,
)
response = query_engine.query("What are the main findings?")

retriever = index.as_retriever(similarity_top_k=5)
nodes = retriever.retrieve("main findings")

Best: set similarity_top_k theo eval, không chỉ theo cảm giác.

Mistake: rebuild index trong request path hoặc không persist index/vector store.

STO

StorageContext

Persist index, docstore, index store, vector store
Khái niệm StorageContext quản lý nơi lưu nodes, index metadata, vector data.
Khi dùng Persist/load index, custom vector DB, production RAG.
Import StorageContext, load_index_from_storage
Methods persist(), from_defaults().
from llama_index.core import (
    StorageContext,
    load_index_from_storage,
)

# Persist
index.storage_context.persist(persist_dir="./storage")

# Load later
storage_context = StorageContext.from_defaults(
    persist_dir="./storage"
)
index = load_index_from_storage(storage_context)

Best: production dùng vector store bền vững và backup storage.

Mistake: chỉ lưu local storage tạm hoặc mismatch embedding model khi load lại.

VS

Vector Stores

External databases for embeddings and metadata filters
Khái niệm Backend lưu vectors/metadata: Chroma, Qdrant, Pinecone, Weaviate, Postgres...
Khi dùng Corpus lớn, persistence, filtering, hybrid search, production scale.
Import Provider-specific vector store + StorageContext.
Methods Dùng qua VectorStoreIndex.from_vector_store() hoặc storage context.
# Example shape; install provider package first.
from llama_index.core import StorageContext, VectorStoreIndex
from llama_index.vector_stores.chroma import ChromaVectorStore

vector_store = ChromaVectorStore(chroma_collection=collection)
storage_context = StorageContext.from_defaults(
    vector_store=vector_store
)

index = VectorStoreIndex.from_documents(
    documents,
    storage_context=storage_context,
)

Best: chọn store có filter, update/delete, backup, tenancy phù hợp dữ liệu.

Mistake: quên metadata filters cho quyền truy cập hoặc multi-tenant data.

QE

Query Engine

Question answering over indexes with retrieval + synthesis
Khái niệm Query engine nhận query, retrieve nodes, synthesize response.
Khi dùng API hỏi đáp RAG đơn giản, Q&A over docs, one-shot answers.
Constructor index.as_query_engine(...), RetrieverQueryEngine.from_args(...)
Methods query(), aquery(), source nodes trong response.
query_engine = index.as_query_engine(
    similarity_top_k=4,
    response_mode="compact",
)

response = query_engine.query(
    "What does the handbook say about refunds?"
)

print(response)

for source in response.source_nodes:
    print(source.score, source.node.metadata.get("source"))

# Async
# response = await query_engine.aquery("...")

Best: expose source nodes/citations; tune top_k + response_mode qua eval.

Mistake: chỉ đọc final answer, không inspect retrieved nodes khi sai.

RT

Retrievers

Fetch relevant nodes before synthesis
Khái niệm Retriever chỉ trả nodes liên quan, không generate answer.
Khi dùng Custom RAG, rerank, inspect recall, compose query engines.
Constructor index.as_retriever(similarity_top_k=...)
Methods retrieve(), aretrieve().
retriever = index.as_retriever(
    similarity_top_k=8,
)

nodes = retriever.retrieve("refund policy")

for n in nodes:
    print(n.score)
    print(n.node.get_content()[:300])

# Combine with rerank/postprocessors
# before response synthesis.

Best: test retriever riêng bằng labelled queries trước khi blame LLM.

Mistake: tăng top_k quá cao khiến synthesis bị loãng và tốn token.

RS

Response Synthesizer

Turn retrieved nodes into final answers
Khái niệm Module tổng hợp answer từ query + retrieved nodes.
Khi dùng Custom retrieval pipeline, thay response mode, citation strategy.
Constructor get_response_synthesizer(response_mode=...)
Modes compact, refine, tree_summarize, no_text.
from llama_index.core import get_response_synthesizer
from llama_index.core.query_engine import RetrieverQueryEngine

synthesizer = get_response_synthesizer(
    response_mode="compact",
)

query_engine = RetrieverQueryEngine.from_args(
    retriever=retriever,
    response_synthesizer=synthesizer,
)

response = query_engine.query("Summarize the contract")

Best: chọn response mode theo tài liệu: short QA, long summary, citations.

Mistake: dùng một response mode cho mọi use case mà không đo quality/cost.

PP

Node Postprocessors

Rerank, filter, trim, reorder retrieved nodes
Khái niệm Postprocessor chỉnh danh sách nodes sau retrieval trước synthesis.
Khi dùng Reranking, metadata filtering, similarity cutoff, context compression.
Import SimilarityPostprocessor, reranker packages.
Parameter node_postprocessors=[...] trong query engine.
from llama_index.core.postprocessor import SimilarityPostprocessor

query_engine = index.as_query_engine(
    similarity_top_k=10,
    node_postprocessors=[
        SimilarityPostprocessor(similarity_cutoff=0.72),
    ],
)

response = query_engine.query("What changed in v2?")

Best: rerank/cutoff giúp giảm context noise khi corpus lớn.

Mistake: cutoff quá cao làm mất recall hoặc không log nodes bị loại.

CE

Chat Engine

Conversational querying over data
Khái niệm Chat engine giữ conversation context và query index theo lượt chat.
Khi dùng Chatbot over docs, support assistant, conversational RAG.
Constructor index.as_chat_engine(chat_mode=...)
Methods chat(), stream_chat(), reset().
chat_engine = index.as_chat_engine(
    chat_mode="context",
    similarity_top_k=4,
    system_prompt="Answer from the docs. Cite sources.",
)

response = chat_engine.chat("What is our refund policy?")
print(response)

stream = chat_engine.stream_chat("Summarize it in bullets")
for token in stream.response_gen:
    print(token, end="")

Best: dùng chat engine khi cần multi-turn; vẫn giới hạn history/context.

Mistake: dùng query engine stateless cho chatbot rồi tự nối history thủ công.

TL

Tools

Functions and query engines exposed to agents
Khái niệm Tools là callable/query engine có metadata để agent chọn dùng.
Khi dùng Agent cần calculator, APIs, search, hoặc query engine chuyên biệt.
Import FunctionTool, QueryEngineTool, ToolMetadata
Methods from_defaults(), callable function với type hints/docstring.
from llama_index.core.tools import (
    FunctionTool,
    QueryEngineTool,
    ToolMetadata,
)

def multiply(a: float, b: float) -> float:
    """Multiply two numbers."""
    return a * b

math_tool = FunctionTool.from_defaults(fn=multiply)

docs_tool = QueryEngineTool(
    query_engine=query_engine,
    metadata=ToolMetadata(
        name="docs_search",
        description="Search internal docs for policy answers.",
    ),
)

Best: tool name/description cụ thể; function có type hints và docstring rõ.

Mistake: agent có quá nhiều tools mơ hồ hoặc tool side effect không guardrail.

AG

Agents

FunctionAgent, AgentWorkflow, ReActAgent, CodeActAgent
Khái niệm Agent dùng LLM chọn tools nhiều bước cho tới khi hoàn thành task.
Khi dùng Model cần quyết định query engine/tool/API nào để gọi.
Import from llama_index.core.agent.workflow import FunctionAgent
Methods Async run(user_msg=...), streaming tùy model.
from llama_index.core.agent.workflow import FunctionAgent
from llama_index.llms.openai import OpenAI

def add(a: float, b: float) -> float:
    """Add two numbers."""
    return a + b

agent = FunctionAgent(
    tools=[add, docs_tool],
    llm=OpenAI(model="gpt-4o-mini"),
    system_prompt=(
        "You answer questions and use tools when needed."
    ),
)

response = await agent.run(
    user_msg="What is 20 + 22? Then check docs for refund rules."
)
print(response)

Best: dùng agent khi cần tool selection; dùng query engine khi task chỉ là QA cố định.

Mistake: cho agent quyết định mọi thứ trong khi retrieval pipeline cần deterministic.

SO

Structured Output

Pydantic models and structured prediction
Khái niệm Ép output theo schema Pydantic thay vì parse text tự do.
Khi dùng Extraction, classification, agent final answer, typed API output.
Import BaseModel; LLM structured predict APIs.
Methods structured_predict(), agent structured output patterns.
from pydantic import BaseModel, Field
from llama_index.llms.openai import OpenAI

class ContactInfo(BaseModel):
    name: str = Field(description="Person name")
    email: str
    company: str | None = None

llm = OpenAI(model="gpt-4o-mini")

info = llm.structured_predict(
    ContactInfo,
    "Extract contact info: Lan, lan@example.com",
)

print(info.email)

Best: schema chặt, descriptions ngắn, validate edge cases.

Mistake: yêu cầu JSON bằng prompt nhưng không validate bằng schema.

WF

Workflows

Event-driven, step-based orchestration for complex apps
Khái niệm Workflow chia app thành steps nhận Event và trả Event kế tiếp.
Khi dùng Branches, loops, async orchestration, human-in-the-loop, agentic RAG.
Import Workflow, step, Event, StartEvent, StopEvent
Methods run(), handler awaitable, stream_events().
from llama_index.core.workflow import (
    Workflow,
    step,
    Event,
    StartEvent,
    StopEvent,
)

class Retrieved(Event):
    context: str

class RAGWorkflow(Workflow):
    @step
    async def retrieve(self, ev: StartEvent) -> Retrieved:
        nodes = retriever.retrieve(ev.question)
        context = "\n\n".join(n.node.get_content() for n in nodes)
        return Retrieved(context=context)

    @step
    async def answer(self, ev: Retrieved) -> StopEvent:
        response = llm.complete(f"Answer from context:\n{ev.context}")
        return StopEvent(result=str(response))

result = await RAGWorkflow(timeout=60).run(question="What changed?")

Best: type events rõ; dùng plain Python cho branches/loops; validate workflow trong tests.

Mistake: nhét workflow phức tạp vào một query engine callback khó debug.

PR

Prompts

Customize templates used by query engines and LLMs
Khái niệm Prompt templates điều khiển synthesis, QA, refine, agents, structured extraction.
Khi dùng Domain-specific tone, citation rules, extraction instructions.
Import PromptTemplate
Methods get_prompts(), update_prompts(), template formatting.
from llama_index.core import PromptTemplate

qa_prompt = PromptTemplate(
    "Context:\n{context_str}\n\n"
    "Question: {query_str}\n"
    "Answer in Vietnamese and cite sources."
)

query_engine.update_prompts({
    "response_synthesizer:text_qa_template": qa_prompt,
})

prompts = query_engine.get_prompts()

Best: giữ prompts ngắn, test bằng eval set, version prompt changes.

Mistake: prompt chống hallucination nhưng retrieval context lại sai hoặc thiếu.

RO

Routers

Route queries to the best tool or query engine
Khái niệm Router chọn query engine/tool phù hợp dựa trên query và metadata.
Khi dùng Nhiều indexes theo domain, summary vs vector search, SQL vs docs.
Import Router query engine/selectors từ llama_index.core.
Pattern Build tools -> selector -> router engine.
from llama_index.core.tools import QueryEngineTool, ToolMetadata
from llama_index.core.query_engine import RouterQueryEngine
from llama_index.core.selectors import LLMSingleSelector

policy_tool = QueryEngineTool(
    query_engine=policy_engine,
    metadata=ToolMetadata(
        name="policy_docs",
        description="Use for HR and refund policy questions.",
    ),
)

router = RouterQueryEngine(
    selector=LLMSingleSelector.from_defaults(),
    query_engine_tools=[policy_tool, api_tool],
)

Best: tool descriptions phải phân biệt rõ domain và dữ liệu.

Mistake: routers chồng routers quá nhiều mà không trace selection.

Khái niệm Index trích entities/relations thành graph để query theo quan hệ.

EV

Evaluation

Measure retrieval and response quality
Khái niệm Eval kiểm tra faithfulness, relevancy, correctness, retrieval quality.
Khi dùng Tuning chunking/top_k/prompts, regression tests cho RAG.
Import Evaluators từ llama_index.core.evaluation.
Pattern Dataset queries -> run pipeline -> evaluator -> compare versions.
from llama_index.core.evaluation import (
    FaithfulnessEvaluator,
    RelevancyEvaluator,
)

faithfulness = FaithfulnessEvaluator(llm=llm)
relevancy = RelevancyEvaluator(llm=llm)

response = query_engine.query("What is the SLA?")

faith_result = faithfulness.evaluate_response(
    response=response,
)
rel_result = relevancy.evaluate_response(
    query="What is the SLA?",
    response=response,
)

Best: tạo eval set nhỏ nhưng thật; track retrieval và generation riêng.

Mistake: tune bằng vài câu hỏi thủ công rồi deploy không regression test.

OBS

Observability

Callbacks, tracing, debugging, token counting
Khái niệm Callback/instrumentation trace retrieval, LLM calls, token usage, latency.
Khi dùng Debug wrong answers, cost tracking, production monitoring.
Import CallbackManager, debug/token handlers, external tracing tools.
Config Set qua Settings.callback_manager.
from llama_index.core import Settings
from llama_index.core.callbacks import (
    CallbackManager,
    LlamaDebugHandler,
)

debug = LlamaDebugHandler(print_trace_on_end=True)
Settings.callback_manager = CallbackManager([debug])

response = query_engine.query("Why did retrieval fail?")

events = debug.get_event_pairs()

Best: log retrieved nodes, prompts, latency, token usage, model/index versions.

Mistake: chỉ log final answer; không có trace thì khó biết lỗi nằm ở retrieval hay synthesis.

PRO

Production RAG

Operational checklist for reliable LlamaIndex apps
Ingestion Incremental, idempotent, versioned transforms, metadata filters.
Query Trace retrieval, postprocess/rerank, return citations.
Storage Durable vector store, backups, delete/update path.
Safety PII handling, tenant filters, eval gates, cost controls.
# Production pattern
1. Load docs with stable IDs and metadata.
2. Transform/chunk/embed in an ingestion job.
3. Persist to vector store + document store.
4. Query via retriever + postprocessors.
5. Synthesize with citations and source nodes.
6. Evaluate retrieval and answer quality.
7. Trace every run and monitor cost/latency.

Best: separate ingestion service from query API; version everything.

Mistake: doing data ingestion, index build, and user query in one request path.

MAP

Quick Decision Map

Pick the right LlamaIndex component
  • SimpleDirectoryReader
    Load local files into Documents quickly.
  • Document / TextNode
    Represent raw data and retrieval chunks.
  • SentenceSplitter
    Chunk long docs before indexing.
  • IngestionPipeline
    Production parsing, transforms, metadata, embeddings.
  • VectorStoreIndex
    Default semantic RAG index.
  • StorageContext
    Persist/load index and use external vector stores.
  • Retriever
    Fetch nodes when you need custom synthesis/rerank.
  • QueryEngine
    One-shot QA over an index.
  • ChatEngine
    Multi-turn conversational RAG.
  • ResponseSynthesizer
    Control how retrieved nodes become answer text.
  • FunctionAgent
    Tool-calling agent for multi-step decisions.
  • Workflow
    Typed, event-driven orchestration with branches/loops.
